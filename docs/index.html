Orchestrated Pipeline Steps

Step 1 — Job Ingestion Signal

A user copies a job board URL into the PrepOps command panel, or clicks "Save to PrepOps" on the Chrome extension. The local server takes the URL and routes it to the scraping queue.

Step 2 — Dynamic Playwright Scrape

For JavaScript-heavy applications, Playwright starts a headless browser locally to await elements. Cheerio loads the DOM snapshot and extracts structural description sections.

Step 3 — AI Matching & Gap Detection

The parsed description markdown is passed to the Gemini API alongside the user's active resume. Gemini runs a structured evaluation to calculate matching scores and missing core skills.

Step 4 — Prep Task Generation

Based on the identified gaps, PrepOps auto-seeds specific, actionable tasks in the candidate's workspace workspace (e.g. "Review Next.js Parallel Routes", "Write custom debounce implementation").

Step 5 — Kanban & Transition Log

As you progress through interview stages, status history records keep a precise log of each transition. If an application remains in a column for over 10 days, a warning dashboard alert lights up.

Deploy PrepOps on your machine in under five minutes. Choose your deployment style below.

Run Dev Server Locally

Perfect for developers who want to inspect changes and run code directly on Node.js.

  1. Install workspace package packages using npm install.
  2. Configure database variables inside .env file.
  3. Build tables and insert mock seed data with Prisma migrations.
  4. Start the Next.js hot-reload development server.
terminal — local dev
$ npm install && cp .env.example .env
$ npx prisma migrate dev
$ npm run dev

Browser Ingestion Extension

Capture postings instantly from any board (LinkedIn, Glassdoor, etc.) and push them directly to SQLite.

  1. Open Google Chrome and head to chrome://extensions.
  2. Enable Developer mode in the top-right toggle.
  3. Click Load unpacked in the top-left section.
  4. Select the extension/ folder in the root directory.
  5. Load any job posting page and click Save To PrepOps.

Gemini API Key Setup

Access high-fidelity matching, gaps analysis, and custom message drafts by configuring your Gemini API variables.

  1. Visit the **Google AI Studio**.
  2. Click **"Get API key"** and create your key credentials.
  3. Paste the key inside the .env file.
  4. Set AI_ENABLED="true" and save.
workspace — .env config
# AI Integration Settings
AI_ENABLED="true"
AI_PROVIDER="gemini"
GEMINI_API_KEY="AIzaSyYourKeyHere..."
GEMINI_MODEL="gemini-2.0-flash"

REFERRAL_TRACKING_ENABLED="true"

System Safeguards

Design Rules & Constraints

Architectural guidelines that ensure local performance, privacy, and consistency.

🔒

Zero Cloud Dependencies

Database state and uploaded files are stored entirely locally, never uploaded to third-party clouds.

👤

Local Resume Records

Resumes are stored on the local filesystem. Only the file path is saved in SQLite to prevent database bloat.

🔑

Secure Secrets Store

Gemini and OpenAI API tokens remain inside `.env` configurations, protected from codebase commits.

🤖

Polite Scraping Checks

The ingestion module reads and respects robots.txt permissions and restricts quick serial request loops.

📂

Structured AI Parsing

AI signals are strictly validated using JSON schemas before database writes to ensure relational safety.

⏱

Transition Timelines

Every column move triggers a `StatusHistory` record to measure performance and capture stale cards.

🆔

UUID Identifiers

Prisma models enforce RFC-compliant UUID keys across all tables to avoid sequential ID attacks.

🎨

Terminal Styling Guides

UI states use Zinc/Slate dark themes with specific Amber, Emerald, and Indigo colors for indicators.
